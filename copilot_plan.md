# תוכנית עבודה: לוקליזציה לעברית ופיתוח צ'אטבוט מלווה חכם (Copilot)

אנחנו רוצים להנגיש את Tandem לבונה העצמאי באמצעות ממשק מלא בעברית, ובמקביל להוסיף **טייס שותף אינטראקטיבי (Copilot Chat Companion)** שישב לצד ממשק העבודה, ילווה את המשתמש בכל שלב, יציע אלטרנטיבות ודעות על אופן היישום, ויפיק פרומפטים מדויקים להעתקה והדבקה בכל פלטפורמה (ולקיחת התוצרים חזרה).

---

## Proposed Changes

### 1. צד השרת (Backend)

#### [NEW] route.ts (mapped to `src/app/api/copilot/route.ts` in code workspace)
נקים נתיב API חדש לקבלת שאלות המשתמש בהקשר הפרויקט:
* קבלת מערך הודעות השיחה (`messages`), מזהה פרויקט (`projectId`), ומזהה משימה פעילה (`taskId` - אופציונלי).
* שליפת ה-PRD, הארכיטקטורה, ההחלטות והסיכומים של הפרויקט מתוך מסד הנתונים.
* קריאה ל-`Vercel AI SDK` עם פרומפט מערכת (System Prompt) מפורט בעברית המגדיר את תפקיד הצ'אטבוט כ-"הטייס השותף של Tandem" (Tandem Copilot).
* המודל יונחה:
  - להסביר בשפה ברורה וידידותית בעברית מה קורה בשלב הנוכחי.
  - להציע פתרונות ודעות טכניות שונות לגבי משימות (למשל: "איך כדאי ליישם את שכבת ה-Auth? בעזרת Supabase Auth המובנה או ספרייה חיצונית?").
  - להפיק ישירות מתוך הצ'אט פרומפטים מובנים מוכנים להעתקה עבור Cursor/Claude/v0.
  - להפיק את ה-**Return Brief Prompt** המדויק שהמשתמש צריך להדביק בכלי ה-AI כדי לקבל את סיכום השינויים שהמערכת מצפה להם.

### 2. צד הלקוח (Frontend)

#### [MODIFY] ProjectClientView.tsx (mapped to `src/app/app/projects/[id]/ProjectClientView.tsx` in code workspace)
* **לוקליזציה לעברית:**
  - תרגום הלשוניות: "Command Center" ➔ "לוח פיקוד", "Product Memory" ➔ "זיכרון מוצר", "Operations" ➔ "משימות פעילות", "Decision Log" ➔ "יומן החלטות", "Risk Register" ➔ "ניהול סיכונים", "Living Docs" ➔ "תיעוד חי".
  - תרגום תיאורי כפתורים, כרטיסי משימות, מצבים ומודלים.
  - תמיכה בכיווניות קריאה (RTL) עבור הטקסטים בעברית לשמירה על חוויית משתמש פרימיום.
* **הוספת פאנל צ'אטבוט צדדי (Tandem Copilot Sidebar):**
  - כפתור צף לפתיחת/סגירת ה-Copilot בראש הדף.
  - חלון צ'אט מעוצב בטכנולוגיית Glassmorphism עם זרימה חלקה של הודעות.
  - **כפתורי פעולה מהירים (Quick Action Chips):**
    * **"מה המשימה הבאה?"**: שולח הודעה אוטומטית המבקשת המלצה ודעות על המשימה הבאה.
    * **"הפק פרומפט משימה"**: מייצר את הפרומפט להדבקה בכלי ה-AI החיצוני (Cursor/Claude/v0) עבור המשימה שנבחרה.
    * **"הפק פרומפט סיום משימה (Return Prompt)"**: מייצר את הפרומפט המדויק להדבקה בכלי החיצוני בסיום העבודה כדי לחלץ את סיכום התוצרים, ההחלטות והסיכונים במבנה markdown מתאים.
    * **"התייעצות על ארכיטקטורה"**: מציע tradeoffs ואפשרויות יישום טכנולוגיות על בסיס ה-Architecture המאושרת.
  - הזנת קלט חופשי על ידי המשתמש.

## Verification Plan

### בדיקות ידניות
1. **ניווט וממשק בעברית:** וידוא שכל הלשוניות, כרטיסי ה-Overview, המודלים והלחצנים מתורגמים לעברית רהוטה וברורה ומיושרים נכון (RTL).
2. **סימולציית צ'אט Copilot:**
   * פתיחת פאנל ה-Copilot.
   * לחיצה על כפתור "הפק פרומפט משימה" ובדיקה שהפרומפט המופק תואם ומכיל את הנחיות ה-PRD.
   * לחיצה על כפתור "הפק פרומפט סיום" ובדיקה שהפרומפט אכן מנחה את ה-AI החיצוני להפיק את ה-Return Brief הנכון.
   * שאילת שאלות חופשיות בעברית (למשל: "איזה tradeoffs יש לי בשימוש ב-Supabase מול Prisma?") ובדיקה שהצ'אטבוט משיב בצורה מקצועיות ומלווה.
3. **קומפילציה:** הרצת `npx tsc --noEmit` לוודא תקינות טיפוסים מלאה.
