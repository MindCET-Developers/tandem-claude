החזון: AI Product OS

המערכת נועדה לעזור למפתחים ויזמים עצמאיים לעבור מרעיון למוצר עובד, באמצעות מתודולוגיית SPDD (Structured Prompt-Driven Development). המערכת שומרת "זיכרון" של הפרויקט, מפיקה פרומפטים מובנים (Mission Briefs) עבור כלי AI (כמו Cursor או Claude), וקולטת את התוצרים כדי לנהל מעקב מבוסס ראיות.

יעדים מרכזיים (MVP)

  • הפיכת רעיון לדרישות מובנות (PRD)
  • שמירת "זיכרון" קבלת החלטות
  • ייצור משימות טכניות
  • שילוב שקוף של כלי AI חיצוניים

מחוץ לתחום (Non-goals)

  • תחליף מלא ל-IDE או Cursor
  • הרצת סוכנים אוטונומיים אוטומטית
  • ניהול הרשאות Enterprise מסובך

התקדמות הפרויקט

ה-MVP נמצא בשלבי סיום. נותרו משימות ליטוש ומקרי קצה.

פירוט שלבי הפיתוח

מה כבר נעשה (Completed)

  • תשתית: Next.js App Router, Tailwind, Supabase Auth & DB.
  • ה-Wizard: קליטת רעיון, הפקת Product Brief ו-PRD אוטומטי מבוסס AI.
  • תכנון משימות: המלצות ארכיטקטורה ופירוק למשימות וייעוד AI Router.
  • מעגל SPDD: יצירת Mission Briefs וקליטת Return Briefs.
  • ממשק: לוח פיקוד (Command Center), יומן החלטות (Decision Log).
  • לוקליזציה: תרגום מלא של הממשק לעברית (RTL).
  • Copilot: צ'אט מלווה חכם שמייצר פרומפטים ישירות מהממשק.

מה נשאר לעשות (To Do / Future Scope)

  • בדיקות E2E: סימולציה מלאה של פתיחת פרויקט ועד סיום עם כלי חיצוני.
  • Drift Detection מתקדם: זיהוי אוטומטי ומדויק יותר של סטיות ב-Return Briefs.
  • פרופיל אישי: אזור Settings מתקדם לניהול מפתחות API פרטיים (BYOK).
  • אופטימיזציית AI: חידוד הפרומפטים כך שלא יאבדו קונטקסט לאורך זמן.
  • (Future Scope) שיתוף פעולה בצוותים (Team Collaboration).
  • (Future Scope) תוספים אינטגרטיביים ישירות לסביבת הפיתוח.

הערימה הטכנולוגית (Tech Stack)

  • Frontend: Next.js App Router, Tailwind CSS, shadcn/ui
  • Backend: Next.js Route Handlers, Supabase (PostgreSQL)
  • AI Layer: Vercel AI SDK (OpenAI / Claude)
  • Authentication: Supabase Auth + RLS
  • Deployment: Vercel

מודל הנתונים (Data Model)

  • Projects: מחזיק את המידע הראשי.
  • Artifacts: שומר גרסאות של PRD, Architecture, Brief (באמצעות JSONB).
  • Tasks: רשימת משימות מסודרת לסטטוסים.
  • Decisions: יומן החלטות המקושר ישירות ל-Artifacts ומשימות.

מוקאפים עיצוביים

הממשק בנוי להיות מודרני, מבוסס "Glassmorphism" ו-Dark Mode כדי להתאים לסביבות הפיתוח המוכרות למפתחים. להלן תצוגה מקדימה המבוססת על קובץ design-proposal.html שיצרנו קודם לכן.
