פרופיל אישי וניהול מפתחות (BYOK)

אפשרות למשתמשים להזין מפתחות API משלהם עבור ספקיות ה-AI (כגון OpenAI ו-Anthropic). זה יאפשר חיסכון בעלויות המערכת ויאפשר למשתמשים לנצל את ה-Tier האישי שלהם.

שינויי Database

הוספת טבלת `user_settings` ל-Supabase לשמירת המפתחות. יש לשקול שימוש ב-pgsodium להצפנת המפתחות ברמת מסד הנתונים כדי לשמור על אבטחה מקסימלית.

אופטימיזציית Drift Detection

שיפור מנגנון זיהוי הסטייה (Drift). כרגע, קליטת Return Briefs יכולה לפספס סטיות עדינות מה-PRD או לבזבז קונטקסט (Tokens) רב מדי של המודל. הצעד הזה ישפר את הזיהוי על ידי חילוץ סמנטי והשוואה נקודתית.

בדיקות וסימולציה מלאה (E2E)

לקראת שחרור ה-MVP, חובה לבצע מעבר אינטגרטיבי על כלל המערכת מקצה לקצה - יצירת יוזר חדש, הפעלת ה-Wizard, אישור ה-PRD, יצירת משימות, שימוש ב-Copilot וקליטת Return Brief פיקטיבי, כדי לוודא שאין צווארי בקבוק או שגיאות ב-Database (כגון RLS או שגיאות Parsing של ה-JSONB).

תוכנית הבדיקה

  • Signup / Login -> הצלחה
  • הזנת "רעיון" חדש ב-Wizard -> הפקת PRD מלא
  • בדיקת Vercel AI SDK Edge Cases (Timeouts / JSON malformed)
  • בדיקת הרשאות (RLS) בין משתמשים שונים במסד הנתונים
